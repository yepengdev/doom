#+TITLE: Palimpsest — 搬移文本而非删除
#+STARTUP: overview

* 用途

在写作/编辑过程中，经常有"这段去掉，但可能以后会用"的犹豫。Palimpsest 提供"搬移而非删除"的三个目的地，让你可以保留文本但不妨碍当前编辑空间：

| 目的地 | 效果 | 持久性 |
|--------+------+--------|
| Buffer 顶部 | 移到当前 buffer 最上方 | 当前会话（保存后随文件持久化） |
| Buffer 底部 | 移到当前 buffer 最下方 | 同上 |
| Trash 文件 | 移到 ~<basename>.trash.<ext>~ 文件 | 跨会话（独立文件持久化） |

* 配置

#+BEGIN_SRC emacs-lisp
(use-package! palimpsest
  :hook ((org-mode . palimpsest-mode))
  :config
  (map! :localleader
        :map org-mode-map
        :prefix ("P" . "Palimpsest")
        :desc "Move to top"    "t" #'palimpsest-move-region-to-top
        :desc "Move to bottom" "b" #'palimpsest-move-region-to-bottom
        :desc "Move to trash"  "T" #'palimpsest-move-region-to-trash))
#+END_SRC

* 操作流程

1. 选中区域（~V~ 行选或 ~v~ 字符选）
2. ~SPC m P t~ — 移至 buffer 顶部
3. ~SPC m P b~ — 移至 buffer 底部
4. ~SPC m P T~ — 移至 ~<file>.trash.<ext>~

Trash 文件格式与原始文件相同，可以用 ~recenter-top-bottom~ 滚动查看，或用 ~find-file~ 打开 trash 文件独立处理。

* ~SPC m P~ 前缀的设计选择

使用 ~P~（Palimpsest 的首字母）而非 ~DEL~ 或其他符号，理由：

- ~SPC m~ 是 Doom 的 localleader（major-mode 本地前缀），~org-mode-map~ 中已有大量命令，~P~ 不会冲突。
- ~t~/~b~/~T~ 对应 ~top~/~bottom~/~Trash~，易于记忆。
- ~T~（大写）与 ~t~（小写）区分目的地（trash vs top）。

* 与 delete/kill 的对比

| 方式 | 效果 | 恢复难度 | 跨会话 |
|------+------+----------+--------|
| ~dd~ / ~x~ | 删除正常文本 | undo（有限步数，关闭后丢失） | × |
| ~d~ ~d~ | 同上 | 同上 | × |
| Palimpsest top | 移到当前 buffer 顶部 | ~gg~ ~d~ ~d~ 后 undo | ✓（在同一文件中） |
| Palimpsest bottom | 移到当前 buffer 底部 | ~G~ ~d~ ~d~ 后 undo | ✓ |
| Palimpsest trash | 移到独立文件 | 打开 ~.trash.~ 文件 | ✓ |

- Palimpsest 不影响 undo 链（不是 deletion —— 是 moving）。
- Trash 文件在 Emacs 会话之间持久保留，关闭文件后不丢失。
- 与 super-save（[[file:super-save.org][super-save.org]]）搭配：palimpsest 移动文本后若 super-save 触发保存，trash 文件也会被保存。

* 与其他写作工具的协同

- [[file:olivetti.org][olivetti.org]] — 提供专注写作界面，palimpsest 在写作流程中处理存疑段落。
- [[file:super-save.org][super-save.org]] — 自动保存，确保 trash 文件不因手动保存遗漏而丢失。
- [[file:evil-insert-hybrid.org][evil-insert-hybrid.org]] — Vim 选择（~V~ / ~v~） + Emacs 编辑键位，palimpsest 只需 ~SPC m P t/b/T~。
