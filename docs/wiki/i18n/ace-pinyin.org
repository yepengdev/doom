#+TITLE: Pinyin 字符跳转 (ace-pinyin)
#+STARTUP: overview

* 用途

[[https://github.com/abo-abo/avy][Avy]] 是 Emacs 的视觉导航工具（~avy-goto-char-timer~，默认按 ~f~、~F~、~t~、~T~）。
~ace-pinyin~ 扩展 Avy，使其支持拼音输入 — 输入 ~xie~ 跳转到对应的中文字符（写、谢、些...）。

* 配置

#+BEGIN_SRC emacs-lisp
(use-package! ace-pinyin
  :commands ace-pinyin-global-mode
  :after-call avy-goto-char-timer
  :init (setq ace-pinyin-use-avy t)
  :config (ace-pinyin-global-mode t))
#+END_SRC

* 延迟加载与 :after-call

| 元素                       | 说明                                            |
|----------------------------+-------------------------------------------------|
| ~:commands~                | 声明 ~ace-pinyin-global-mode~ 为加载触发器       |
| ~:after-call~              | Doom 提供的 use-package 扩展，指定 Emacs 命令被调用后加载 |
| ~avy-goto-char-timer~      | Avy 的字符跳转命令（~f~ / ~F~ / ~t~ ~T~）      |

~:after-call~ 的加载逻辑：

1. 用户第一次按 ~f~（触发 ~avy-goto-char-timer~）
2. 运行完成后，Doom 标记 ~ace-pinyin~ 可加载
3. Emacs idle 时自动加载 ~ace-pinyin~
4. 后续 ~f~ 操作将使用拼音增强的 Avy

这种策略避免了在启动时加载 ace-pinyin 及其拼音映射表（~100KB+ 的哈希表）。

* ace-pinyin-use-avy

#+BEGIN_SRC emacs-lisp
(setq ace-pinyin-use-avy t)
#+END_SRC

| 值   | 行为                                                             |
|------+------------------------------------------------------------------|
| ~t~  | 将 ace-pinyin 的拼音映射集成到 Avy 的字符提示中（推荐）            |
| ~nil~ | 替换 Avy 的默认提示（纯拼音模式，不显示汉字）                    |

~ace-pinyin-use-avy~ = ~t~ 时，Avy 的字符提示行为：

1. 按 ~f~ → Avy 开始监听字符输入
2. 输入 ~xie~ → Avy 在所有匹配 ~xie~ 音节的汉字字符上放置提示标记
3. 提示标记（如 ~a~、~b~、~c~...）覆盖在对应字符上
4. 按提示标记跳转到目标位置

同时，输入 ~写~（汉字字面）也会被 Avy 识别（原始 Avy 行为保持）。

* 使用场景

| 操作                 | 结果                                               |
|----------------------+----------------------------------------------------|
| 中文文本中按 ~f~    | Avy 字符跳转：输入 ~xie~ → 匹配所有 ~xie~ 音的汉字  |
| 按 ~t~（avy-goto-char-timer） | 同上，支持拼音                               |
| 按 ~F~（大写的字符跳转）     | 同上（跳转到行首的匹配字符）                    |
| 中文长段落导航       | 输入 ~zhong yao~ → 跳转到"重要"的"重"或"要"       |
| 代码中中文注释导航   | 输入 ~shuo ming~ → 跳转到"说明"                     |

不需要切换到英文输入法来导航中文文本 — 拼音和汉字都是 ASCII 输入。

* 拼音映射表

~ace-pinyin~ 内部维护从拼音到汉字的映射表：

: ("a" → "阿啊呵...")
: ("ai" → "爱挨矮...")
: ...
: ("zuo" → "做左坐座作...")

映射表在 ~ace-pinyin-global-mode~ 启用时构建。
~:after-call~ 延迟加载确保此构建不在启动时发生。

* 与 evil-pinyin 的区别

| 工具            | 领域               | 用途                           |
|-----------------+--------------------+--------------------------------|
| ~evil-pinyin~   | 搜索（Orderless）  | 拼音匹配文件名/命令/缓冲区内容  |
| ~ace-pinyin~    | 导航（Avy）        | 拼音跳转到可视范围内的字符位置  |
| ~~              | ~~                 | ~~                             |
| 输入 ~xie~ 在   | Vertico 搜索列表   | 匹配所有含"写/谢/些"的文件名   |
| 输入 ~xie~ 在   | 当前缓冲区（Avy）  | 跳转到"写/谢/些"字符的位置     |

两者互补：~evil-pinyin~ 负责找文件，~ace-pinyin~ 负责在文件中定位。

* 注意事项

- ~ace-pinyin~ 的拼音映射表只覆盖常见汉字（约 6763 个 GB2312 汉字外加常用字）
- 生僻字（如 𪚥、𠄳）可能无拼音映射
- 多音字（如"行"→ xíng / háng）两种读音都被映射，输入任一都可匹配
- 提示标记的候选数量受 Avy 限制（默认最多 256 个候选，超出部分无法显示）
