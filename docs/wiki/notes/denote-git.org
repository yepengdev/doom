#+TITLE: Git 自动提交

* 用途

每次 ~denote-after-new-note-hook~ 触发时自动执行 ~git add -A && git commit~。
作为安全网——即使忘记手动提交，所有新建笔记也不会丢失。

* 配置

#+BEGIN_SRC emacs-lisp
(defun my/denote-git-auto-commit ()
  (when-let ((dir (denote-directory)))
    (let ((git-dir (expand-file-name ".git" dir)))
      (when (file-exists-p git-dir)
        (let ((default-directory dir))
          (unless (zerop (call-process "git" nil nil nil "add" "-A"))
            (message "WARN: denote git add failed"))
          (unless (zerop (call-process "git" nil nil nil
                                       "commit" "--allow-empty"
                                       "-m" "auto: note saved"))
            (message "WARN: denote git commit failed")))))))
(add-hook 'denote-after-new-note-hook #'my/denote-git-auto-commit)
#+END_SRC

* 设计决策

| 决策 | 理由 |
|------+------|
| ~(when-let ((dir (denote-directory)))~ | 保证 ~denote-directory~ 已设置且非空 |
| ~(file-exists-p git-dir)~ 检查 | 笔记目录可能还不是 git 仓库（如首次配置时）|
| ~call-process~ 非阻塞 | 不用 ~shell-command~（避免弹出 *Async Shell Command* buffer）|
| ~nil nil nil~ 参数 | 不等待输出、不显示 buffer、不重定向 stdin |
| ~(unless (zerop ...) (message "WARN: ..."))~ | 检查返回值，~git~ 命令失败时在 minibuffer 给出警告（而非静默失败）|
| ~--allow-empty~ | 即使没有新文件变更也允许 commit（多次创建笔记后手动 commit 了也可以再次 commit）|
| commit message ~"auto: note saved"~ | 统一前缀，方便 ~git log --grep="auto:"~ 过滤自动提交 |

* 工作流程

1. 用户执行 ~denote~ 创建新笔记（或任何调用 ~denote-after-new-note-hook~ 的操作）
2. ~my/denote-git-auto-commit~ 被调用
3. 检查 ~~/org/denote/.git/~ 是否存在
4. ~git add -A~ 暂存所有变更（不限于新文件，也包含已修改文件）
5. ~git commit --allow-empty -m "auto: note saved"~ 提交
6. 任一命令非零退出时 ~message~ 输出警告

* 局限性

- 不处理并发冲突（两个 Emacs 实例同时创建笔记可能导致提交失败）
- 不处理网络（无 ~git push~，只做本地 commit。推送由用户手动或后台 ~git push~ 定时任务处理）
- 不覆盖已有笔记的保存（编辑已有笔记的 ~after-save-hook~ 由 editor/super-save 处理，本钩子只处理新建）
- 目录必须已经是 git 仓库（需要在首次使用前 ~git init~ 一次）
