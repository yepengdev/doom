#+TITLE: 序列笔记 (denote-sequence)

* 用途

Folgezettel（德语："序列卡片"）——一种介于纯层级（目录树）和纯平铺（tag 系统）之间的中间方案。
通过 SIGNATURE（签名）字段建立笔记的父子兄弟关系：

#+BEGIN_SRC text
1                                   项目
├── 1.1                             架构设计
│   ├── 1.1.a                       数据库选型
│   └── 1.1.b                       API 设计
└── 1.2                             部署方案
    └── 1.2.a                       Docker 配置
#+END_SRC

核心思想：

- **父子关系**：~1.1~ 是 ~1~ 的子级，~1.1.a~ 是 ~1.1~ 的子级
- **兄弟关系**：~1.1.a~ 和 ~1.1.b~ 是同级的兄弟节点
- **层级增长**：可以在任意层级插入新节点，~1.1.1~、~1.1.2~ 等自动按序生成
- **签名与命名独立**：SIGNATURE 存储在文件 front-matter（~#+filetitle:~ 等的扩展字段）和文件名中，不影响 Denote 的 ID/标题/关键词体系

* 配置

#+BEGIN_SRC emacs-lisp
(use-package! denote-sequence
  :after denote)
#+END_SRC

不需要额外 ~:custom~ 设置，~denote-sequence~ 使用 ~denote~ core 的配置（签名方案通过用户交互选择）。

* 操作

** 新建序列笔记（~SPC r d s~）

1. 输入父笔记的 ID/标题（或选择已存在的序列笔记）
2. 选择或输入子级的签名（如 ~1.1.a~，自动递增建议）
3. 输入标题和关键词（同普通 ~denote~ 流程）
4. 生成的笔记文件名包含签名

** 在 explore 网络图中查看层级

~SPC r d e n~（网络图）可切换至 ~sequence~ 视图模式，以树状图显示层级关系。

** 动态块

在 Org 文件中通过 ~C-c C-x x~ 插入 ~denote-sequence~ 动态块，展开为层级列表：

#+BEGIN_SRC org
#+BEGIN: denote-sequence :regexp "项目"
#+END:
#+END_SRC

刷新（~C-c C-x C-u~）后显示匹配笔记的序列层级树。

* 签名方案

| 方案 | 格式 | 示例 | 适用场景 |
|------+------+------+----------|
| numeric | 1, 1.1, 1.1.1 | ~1.2.3~ | 严格层级结构，适合技术文档 |
| alphanumeric | A, A1, A1a | ~B3c~ | 章节划分，适合教材笔记 |
| alphanumeric-delimited | A-1, A-1-a | ~B-3-c~ | 可读性更好，适合长序列 |

签名方案的选择会影响网络图的 ~sequence~ 视图排序和动态块的渲染顺序。
通过 ~denote-sequence--scheme~ 交互选择。

* 与 Denote 体系的关系

- 序列签名存储于文件 front-matter 中的 ~SIGNATURE:~ 字段
- 签名也会写入文件名（在标题之前或之后），但由 Denote 的命名规则统一管理
- 签名不影响 ~consult-notes~ 搜索、~denote-find-link~ 等核心功能
