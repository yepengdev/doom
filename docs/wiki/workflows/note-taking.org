#+TITLE: 笔记管理流

* 阶段 1：创建

| 场景 | 方法 |
|------+------|
| 今日日记 | ~SPC r d j~ |
| 知识笔记 | ~SPC r d n~ → 输入标题 + 选关键词 |
| 指定日期 | ~C-u SPC r d d~ |
| 序列笔记 | ~SPC r d s~ → 选父笔记 + 输入签名 |
| 快速 fleeting note | ~SPC n n~ → Deft 目录 |

* 阶段 2：链接与关联

| 操作 | 方法 |
|------+------|
| 链接到已有笔记 | ~SPC r d i~ → 输入标题 |
| 链接到标题级别 | ~SPC m D h~ → 选文件 → 选标题 |
| 查看反向链接 | ~SPC m D b~ |
| 查看标题级别反向链接 | ~SPC r d H~ |

* 阶段 3：组织与分类

| 方法 | 说明 |
|------+------|
| 关键词筛选 | ~SPC r d m~ → ~/ k~ |
| 重命名关键词 | ~SPC r d k~ |
| 按关键词/正则筛选 | ~SPC r d m~ → ~/ r~ ~/ k~ ~/ o~ |
| 提取子树为独立笔记 | ~SPC m D x~ |

* 阶段 4：发现与探索

| 操作 | 说明 |
|------+------|
| ~SPC r d e c~ | 查看笔记总数 |
| ~SPC r d e C~ | 关键词频次 |
| ~SPC r d e r~ | 随机笔记（发现自己写过的但忘了的内容） |
| ~SPC r d e n~ | d3.js 网络图 |
| ~denote-menu-menu~ 筛选 | 符合条件的笔记列表 |

* 阶段 5：维护

| 操作 | 说明 |
|------+------|
| Git 自动提交 | 每次 ~denote-after-new-note-hook~ 自动 ~git commit~ |
| 死链接检查 | ~denote-explore-check-dead-links~ |
| 孤立笔记 | ~denote-explore-check-orphan-notes~ |
| 重复 ID | ~denote-explore-check-duplicate-identifiers~ |
